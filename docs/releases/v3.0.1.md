# Xendivel v3.0.1

## Highlights

- Reverted invoice PDF generation to Spatie Browsershot.
- Removed the Typeset.sh subscription and private Composer repository requirement.
- Kept Laravel 13, PHP 8.5, Pest 4, and Orchestra Testbench 10/11 support from v3.0.0.

## Changed

- Invoice PDFs are rendered with `spatie/browsershot:^5.2.3`.
- `Invoice::make()`, `save()`, `download()`, `template()`, `paperSize()`, `orientation()`, and `fileName()` remain unchanged.
- `xendivel.typesetsh.*` config has been replaced by `xendivel.browsershot.*`.
- Default invoice paper size remains A4; supported sizes remain A4, Letter, and Legal.

## Migration

1. Upgrade to `glennraya/xendivel:^3.0`.
2. Remove any Typeset.sh repository/auth setup that was added only for Xendivel.
3. Install Puppeteer and Chrome in the root Laravel application:
   - `npm install puppeteer`
   - `npx puppeteer browsers install chrome`
4. Publish or merge updated `xendivel` config (`xendivel-config`).
5. Set Browsershot binary/path options only when your deployment environment needs them.

## Verification Notes

- Package metadata should pass `composer validate --strict`.
- Full package suite should pass with `composer test`.
- Focused PDF behavior is covered by `tests/Unit/InvoicePdfTest.php`.
- Browser-backed PDF tests skip when Node/Puppeteer/Chrome are unavailable.

## Dependency Deltas (v3.0.0 -> v3.0.1)

- Added: `spatie/browsershot:^5.2.3`.
- Dropped: `typesetsh/typesetsh:^0.27.0`.
- Dropped: `typesetsh/font-noto-cjk-sub:^1.0`.
