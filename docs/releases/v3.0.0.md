# Xendivel v3.0.0

## Highlights

- Official Laravel 13 support.
- PDF rendering now uses `typesetsh/typesetsh` core directly.
- New resolver configuration surface for predictable asset loading in custom invoice templates.

## Breaking Changes

- Removed dependency on `typesetsh/laravel-wrapper`.
- Dropped support for `spatie/browsershot`-based PDF generation.
- Teams with custom templates should review and tune `xendivel.typesetsh.*` settings.

## Migration

1. Upgrade to `glennraya/xendivel:^3.0`.
2. Keep Typeset repository/auth configured in the root app:
   - `composer config repositories.typesetsh composer https://packages.typeset.sh`
   - `composer config -g http-basic.packages.typeset.sh "{PUBLIC_ID}" "{TOKEN}"`
3. Publish or merge updated `xendivel` config (`xendivel-config`).
4. Validate custom template resources against:
   - `typesetsh.allowed_directories`
   - `typesetsh.allowed_protocols`
   - `typesetsh.base_dir`
   - `typesetsh.cache_dir`
   - `typesetsh.timeout`
   - `typesetsh.download_limit`

## Verification Notes

- Package metadata validated with `composer validate --strict`.
- PDF behavior validated with focused package test suite:
  - `tests/Unit/InvoicePdfTest.php`
- Added coverage for:
